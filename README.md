# DQN-based Dynamic Spectrum Sharing for O-RAN near-RT RIC

A Deep Q-Network (DQN) agent for dynamic Physical Resource Block (PRB)
allocation in a simulated O-RAN near Real-Time RAN Intelligent Controller
(near-RT RIC) environment.

## Overview

Five User Equipment (UE) nodes report Channel Quality Indicators (CQI) and
current throughput every time step. The DQN agent observes this 10-feature
network state and outputs a PRB allocation vector that maximises aggregate
network throughput subject to a per-UE minimum fairness constraint. Results
are compared against round-robin and proportional-fair baseline schedulers
across three traffic scenarios.

## Training

```bash
# Train on the default uniform scenario (10,000 steps)
python train.py

# Train on all three scenarios
python train.py --scenario uniform
python train.py --scenario heterogeneous
python train.py --scenario bursty

# Custom options
python train.py --scenario bursty --total-steps 20000 --lr 5e-4
```

Training prints step, epsilon, average reward, and loss every 100 steps.
The trained model is saved to `models/dqn_trained_<scenario>.pt`.

## Architecture

### Environment (`env/ran_environment.py`)

| Parameter | Value |
|-----------|-------|
| UEs | 5 |
| Total PRBs | 50 |
| Episode length | 200 steps |
| State | `[cqi_0..4, prb_0..4]` — 10 floats |
| Action | PRB weight vector — 5 floats (normalised to sum to 50) |
| Reward | Total throughput − 5 × fairness shortfall |
| Throughput | `prbs × log2(1 + (cqi−1)×2)` |

Three scenarios control CQI initialisation and evolution:
- **Uniform** — all UEs similar, slow Gaussian drift
- **Heterogeneous** — UEs split across high/medium/low tiers with mean reversion
- **Bursty** — normal drift with 10% per-step spike probability

### DQN Agent (`agent/dqn_agent.py`)

```
Input (10) → Linear(128) → ReLU → Linear(128) → ReLU → Linear(5) → Softplus
```

- **Optimizer**: Adam, lr=1e-3
- **Loss**: Huber (smooth L1)
- **Target network**: hard-copied every 100 gradient steps
- **Exploration**: ε-greedy, ε decays 1.0 → 0.05 over training
- **Gradient clipping**: max norm 10

### Replay Buffer (`agent/replay_buffer.py`)

Fixed-size circular buffer (`collections.deque`) of capacity 10,000.
Stores `(state, action, reward, next_state, done)` tuples.
Uniform random sampling for training batches.
